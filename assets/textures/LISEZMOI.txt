TEXTURES PBR SEAMLESS (pour embellir le jeu)
============================================

Chaque surface peut recevoir un JEU de 3 images PBR (toutes optionnelles).
Le NOM est celui de la surface (voir liste plus bas) + un SUFFIXE selon le
type de carte :

    <nom>_B.png   BaseColor  (la couleur / albedo)   -> ce qu'on voit
    <nom>_R.png   Normal     (le relief)             -> eclairage / volume
    <nom>_P.png   Packed     (AO / Rugosite / ...)   -> ombrage (canal R = AO)

Exemple pour l'herbe :  grass_B.png  grass_R.png  grass_P.png

A deposer dans  assets/textures/  (ou .jpg / .jpeg).

COMMENT CA MARCHE
-----------------
- BaseColor (_B) presente  -> elle habille la surface ; sinon couleur plane de
  repli (le jeu reste joli meme sans aucune image).
- Au moins une carte Normal (_R) presente -> un ECLAIRAGE PAR RELIEF s'active
  automatiquement (les creux/bosses reagissent a la lumiere). Sans aucune carte
  Normal, l'eclairage ne s'active pas (rendu identique a avant, zero risque).
- Packed (_P) : le canal ROUGE est utilise comme occlusion ambiante (assombrit
  les creux). Les autres canaux sont reserves (rugosite/metallique).
- Compat : un ancien fichier  <nom>.png  (sans suffixe) est lu comme BaseColor.

----------------------------------------------------------------
NOMS DE SURFACES (a combiner avec _B / _R / _P)
----------------------------------------------------------------
Sols / terrain :
    grass              herbe proche (plaine, sol vu d'en haut)
    grass_far          herbe lointaine (crete a l'horizon)
    forest_floor       sol de foret proche
    forest_floor_far   sol de foret lointain
    rock               roche (pente de montagne)
    rock_dark          roche en bas de pente (plus sombre)
    water              eau (lac)
    sand               rive / terre au premier plan

Vegetation :
    bark               ecorce (troncs d'arbres)
    foliage            feuillage (arbres, buissons)

Personnage :
    skin               peau (mains du joueur)

----------------------------------------------------------------
CONSEILS IMPORTANTS
----------------------------------------------------------------
- Repetition PROPRE : images CARREES en PUISSANCE DE 2 (256x256 ou 512x512),
  "seamless" (bords qui se raccordent). Important sur telephone.
- Les 3 cartes d'une meme surface doivent avoir LA MEME taille / le meme cadrage
  (elles se superposent pixel par pixel).
- La carte Normal doit etre au format "OpenGL" classique (bleu dominant).
- Noms EXACTS (minuscules, sans accents) + bon suffixe (_B / _R / _P).
- Apres ajout : pousse le projet (envoyer-sur-github.bat) pour inclure les
  images dans l'APK.

Reglages (direction de la lumiere, activation) : src/widgets/pbr.py
Correspondance noms <-> surfaces + couleurs de repli : src/widgets/textures.py
